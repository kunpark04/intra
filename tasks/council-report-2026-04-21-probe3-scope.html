Probe 3 Scope — Session-Structure Sweep on Combo 865 @ 1h

2026-04-21 UTC · 5 advisors · 5 peer reviews · 1 chairman · Probe 2 §5.6 binding

The Question

Probe 2 PASSED on 1h (net Sharpe 2.89, +$124,896/yr, 220 trades on held-out bars). §4 authorizes Probe 3 (Option Y session-structure sweep) on the test-passing timeframe. Probe 2 §5.6 requires a fresh council before Probe 3 signing.

6 scope axes to decide:

  1. Session-filter set (narrow ≤4 vs. wide 7+)
  2. Parameter holdover (freeze all 28 vs. ±δ neighborhood)
  3. Timeframe lock (1h only per §4)
  4. Trade-count floor (post-filter n_trades_min)
  5. Regime-window stress test (temporal halves — HARD gate or diagnostic?)
  6. Sizing + friction model (flat $500 / $5 RT — locked?)

The Recommendation Narrow probe, multiplicity-gated

Run First Principles' three-stack, not Expansionist's wide grid. Probe 3 is not a confirmatory session sweep — it is an exit ritual for a combo that survived a post-hoc selection of ≥9,000 effective tests and now faces a proper multiplicity-corrected gate it may already fail.

  1. Scope: 1h only, freeze all 28 non-session params. 4 session buckets {all / RTH / RTH-lunch-exclude / overnight}. 15m included as hard-fail negative control.
  2. Multiplicity-corrected Sharpe floor: Bonferroni-adjusted over full decision-tree N, not the naive 1.3 gate. Commit the exact number pre-probe.
  3. Regime halves as HARD gate: both halves of the test window must clear the corrected gate on any passing bucket. One-half-only-passing = FAIL.
  4. Sizing: flat $500/trade, $5/contract RT friction (locked per user directive).
  5. Preregistered Option Z sunset trigger: failure retires the combo-865 carve-out and sunsets Z-score family on NQ/MNQ. In writing.
  6. Preregistered decommission trigger: if paper-trading authorizes, live Sharpe < 0 on N=50 fresh trades halts live.
  7. Test>train bootstrap: compute P(test_Sharpe > train_Sharpe | H0) on observed (2.89, 2.098) × (220, 747). Report in §4.

Where the Council Agrees

  • Parameter freeze. 4/5 (all but Expansionist). All 28 non-session params from combo 865 stay fixed. No neighborhood jitter. §5.2 of Probe 2 preregistration carries forward.
  • Timeframe lock at 1h. 4/5 explicit; Expansionist's 1m/5m proposal violates §7.6 terminal from Probe 1 and is ruled inadmissible.

Where the Council Clashes

Upside discovery vs. falsification discipline

  • Expansionist: Combo 865 is the strongest clean result in project history; Probe 3 should expand the search (7 buckets + neighborhood + 1m/5m + Kelly). Starving the one positive result with falsification discipline throws away discovery potential.
  • Contrarian / First Principles / Outsider / Executor: 865 is a post-hoc survivor of ≥9,000 effective tests. Probe 3's role is to test whether the Probe 2 PASS survives adversarial gates (multiplicity, regime, decommission). Expanding the search compounds the degrees-of-freedom problem the sunset was already engineered to close.

15m inclusion — live candidate vs. hard-fail negative control

  • Expansionist: 15m stays as a live second-TF candidate — session filtering might unlock it.
  • Outsider / Contrarian: 15m as hard-fail negative control. If any 15m bucket passes where 1h doesn't → bucket-level artifact → Probe 3 invalid.
  • Executor: Drop 15m entirely (compute savings; it failed §4 mechanically).
  • Chairman resolution: Keep as hard-fail negative control (D/E framing wins; costs ~2s compute; high falsification leverage).

Blind Spots the Council Caught (Peer Review)

What ALL five advisors missed

  • Multiplicity-corrected Sharpe floor was not computed. At α=0.05 Bonferroni over ~9,000 sweep tests on 220-trade samples, required Sharpe ≈ 3.7. Combo 865 at 2.89 already fails a proper multiplicity-corrected gate. Probe 3 cannot use 1.3.
  • Effective N denominator needs the full decision tree — 3×3000 sweep tests + ML#1 ranker + cross-TF filter + Probe 2 §4 gate ≈ 20,000+ effective tests. Multiplicity denominator is larger than any advisor wrote down.
  • Posterior P(real edge) not preregistered. Under a reasonable prior and the multiplicity adjustment, P(real edge | Probe 2 PASS) ≈ 2-3%. Probe 3 needs a posterior gate (paper-trading requires ≥ 0.5) to know what a PASS is worth.
  • ML#2 V3/V4 precedent mandates a decommission trigger, not just a ship gate. Both V3 (shipped Sharpe 1.78) and V4 (shipped 2.13) failed combo-agnostic audits after clean OOS. Probe 3 must preregister paper-trading decommission conditions.
  • Test>train Sharpe inversion has a computable null probability. Bootstrap P(test_Sharpe > train_Sharpe | H0) on observed (2.89, 2.098) × (220, 747). If < 1%, diagnostic of regime favorability or selection. Preregister as observation or gate.

Biggest individual blind spot — The Expansionist (5/5 reviewers)

  • Treats combo 865 as discovered alpha rather than post-hoc survivor of Berkson selection.
  • 1m/5m proposal directly violates §7.6 terminal of Probe 1 — re-opens the bar-TF axis for a single combo, re-multiplying the searched space the sunset was specifically closing.
  • Kelly-fractional "as diagnostic, not for live sizing" is a number-that-pulls-decisions — it presupposes the edge is real, which is exactly what Probe 3 is supposed to test.
  • Misses the ML#2 V3/V4 revocation precedent: confirmatory evaluation on narrow success metrics is the pattern that already cost the project two shipped stacks.

Advisor Responses (Full Text)

The Contrarian — Narrow probe, multiplicity-first, preregistered sunset trigger

Combo 865 at 1h is a post-hoc survivor of a 13,814-combo sweep. Test-partition net Sharpe 2.89 on 220 trades sounds impressive, but the null distribution of the maximum Sharpe from 13,814 independent draws has a large upper tail — and 220 trades is a small sample where Sharpe estimator noise alone produces ±0.5 swings.

Before any session probe runs, the honest Probe 3 design should preregister a multiplicity-corrected Sharpe floor, not the naive 1.3 gate. At α=0.05 with Bonferroni over ~9,000 effective tests (post-trade-count gate), the required Sharpe on 220 trades is closer to 3.5–3.8. Combo 865 already fails the proper corrected gate before Probe 3 starts. Probe 3's role should be falsification, not confirmation.

Recommendations: ≤4 session buckets; freeze all 28 non-session params; regime halves as HARD gate (both halves must pass the corrected floor); 15m as mandatory negative control; preregistered Option Z sunset trigger in writing before signing.

The First Principles Thinker — Reframe the probe as an exit ritual

The framing "session-structure sweep to confirm combo 865's edge" is already wrong. Combo 865 survived (a) a 13,814-combo sweep, (b) an ML#1-driven ranker, (c) a cross-TF consistency check, (d) a 2-gate §4 readout on one window. Each filter already concentrated the posterior; adding a 7-bucket session sweep without a matching posterior update is ceremony, not science.

The real question: "Given combo 865's 1h PASS on one held-out window, what is P(real edge) and what is the minimum additional evidence that moves that posterior past the paper-trading threshold?"

That needs three orthogonal probes, not a session sweep: (1) multiplicity-adjusted inference — commit the Bonferroni-corrected floor pre-probe; (2) temporal halves as primary evidence — not a side test; (3) parameter-neighborhood stability — does 865's edge survive ±5% jitter? Hard-sparse edges diagnose overfitting; stable neighborhoods diagnose signal. Session buckets are supplementary after gate-passing. Flat $500 sizing. Preregister sunset trigger. 15m as negative control, not upside probe.

The Expansionist — Wide session grid + parameter neighborhood + 1m/5m probes

Combo 865 at 1h on the test window delivered Sharpe 2.89, $124k/yr, 220 trades — after friction, on held-out bars, with a pre-registered gate. That is the single strongest clean result this project has ever produced. The council is one peer-review round away from drafting a probe that treats it like a suspect rather than a lead.

Probe 3 should be designed for upside discovery: (1) 7 session buckets — all / RTH-full / RTH-morning / RTH-afternoon / RTH-lunch-exclude / overnight-only / Asia+Europe; (2) parameter neighborhood around 865, ±5% on continuous and ±1 bar on discrete windows (~40 combos × 4 buckets); (3) 1m and 5m probe on 865's parameter dict — §7.6 sunsets the family at intermediate bar-TFs, but 865's carve-out is explicitly a single-realization exception; (4) Kelly-fractional diagnostic on 220 trades — if f* < 0.02 edge is thin, if f* > 0.15 case is strong.

15m stays in Probe 3 as a live second-TF candidate, not just a negative control. Flat $500 sizing is fine. Preregister an Option Z sunset trigger if the user wants one, but don't let falsification-discipline optimism-starve the one positive result we have.

The Outsider — Freeze everything, structure for an outside reader

I'm reading Probe 2 for the first time. What I see: one combo, one timeframe, one held-out window, PASS — with a footnote that test Sharpe exceeds train Sharpe, which is the classic regime-window artifact pattern.

A reader who didn't grow up with this project would ask: (1) "Is it stable in time?" — test>train is a red flag, split the window in halves before session sweeps; (2) "Is it stable in parameters?" — 28 knobs, 865 picked because it PASSED in a 13,814-combo sweep, so freeze the knobs and test SESSION only; (3) "What does a FAIL look like, and who authorizes it?" — mechanical decision rule same as Probe 1 §3 and Probe 2 §4, including a sunset trigger given the V3/V4 precedent.

Recommendations: ≤4 session buckets; freeze all 28 non-session params; regime halves as HARD gate; 15m included as negative control; flat $500 sizing; preregister Option Z sunset trigger in §4.

The Executor — Four buckets, one week, done

Probe 2 took 2 seconds of compute on sweep-runner-1 and 30 minutes of writeup. Probe 3 can be the same shape if scope is disciplined.

Scope: 1h only, freeze all 28 non-session params, drop 15m entirely (it failed §4 mechanically, keeping it for negative-control value costs a bucket and adds noise), 4 session buckets, flat $500 sizing with $5/contract RT friction.

Execution: reuse scripts/param_sweep.py --start-combo 865 --end-combo 866 with a new --session-filter override flag. One combo × 4 buckets = 4 runs, ~2s each. Wrapper + readout scripts mirror Probe 2's.

Gates: per-bucket net_sharpe ≥ 1.3, n_trades ≥ 50, net_$/yr ≥ $5000. For each passing bucket, also split window in halves as a diagnostic (not a gate — hard halves double thresholds and kill live buckets).

Timeline: prereg half-day, signing half-day, remote run 10 min, readout 30 min, verdict 30 min. Total ~1.5 days. Don't add param neighborhoods, 1m/5m, Kelly — not preregistered, distracting. One probe, one answer, re-council.
